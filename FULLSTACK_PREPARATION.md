### 2.2 通信协议方案

#### 方案选择：REST API (主) + WebSocket (辅)

**REST API** - 用于常规 CRUD 操作:
- 统一使用 `http://localhost:3001/api` 前缀
- 响应格式标准化
- 错误处理统一

**WebSocket** - 用于实时通知:
- 连接地址：`ws://localhost:3001/ws`
- 用于推送实时数据更新

### 2.3 数据模型对齐

#### 批量操作模块

**后端响应格式**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "items": [
      {
        "id": 1,
        "title": "任务标题",
        "status": "pending",
        "created_at": 1712822400000
      }
    ],
    "total": 100,
    "page": 1,
    "page_size": 20
  }
}
```

**前端 Store 处理**:
```typescript
// src/stores/batch.ts
interface BatchItem {
  id: number;
  title: string;
  status: 'pending' | 'completed' | 'failed';
  createdAt: number;
}

interface BatchState {
  items: BatchItem[];
  selectedIds: number[];
  total: number;
  loading: boolean;
}
```

### 2.6 联调问题排查清单

| 问题类型 | 排查步骤 | 解决方案 |
|---------|---------|---------|
| CORS 错误 | 检查后端 CORS 配置 | 添加允许的前端域名 |
| 认证失败 | 检查 JWT token 格式 | 确认 token 正确传递 |
| 数据格式不匹配 | 对比前后端数据模型 | 添加适配器层 |
| 跨域 WebSocket | 检查 WS 连接配置 | 使用同源策略 |
| 时区问题 | 确认时间格式 | 统一使用 UTC + 前端转换 |
