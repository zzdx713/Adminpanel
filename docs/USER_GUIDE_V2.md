## 系统简介

### 什么是 OpenClaw-Admin？

OpenClaw-Admin 是一个基于 Vue 3 + TypeScript 构建的现代化 AI 智能体管理平台，为 OpenClaw Gateway 提供完整的 Web 管理界面。通过直观的可视化操作，用户可以轻松管理 AI 智能体、会话、模型、频道、技能等核心功能。

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3.5 + TypeScript + Vite |
| UI 框架 | Ant Design Vue 3.x |
| 状态管理 | Pinia |
| 路由 | Vue Router 4.x |
| HTTP 客户端 | Axios |
| 后端 | Node.js + Express |
| 数据库 | SQLite (better-sqlite3) |
| 认证 | JWT + bcrypt |

### 核心功能

## 快速入门

### 1. 系统安装

#### 方式一：Docker 部署（推荐）

```bash
# 克隆项目
git clone https://github.com/itq5/OpenClaw-Admin.git
cd OpenClaw-Admin

# 启动服务
docker-compose up -d

# 访问系统
open http://localhost:10001
```

#### 方式二：本地部署

```bash
# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，配置 Gateway 地址等

# 启动开发服务器
npm run dev

# 生产构建
npm run build
npm run start
```

### 2. 系统登录

#### 访问系统
打开浏览器，访问 `http://localhost:10001`（或您的服务器地址）。

#### 首次登录
1. 输入用户名（默认：`WKP`）
2. 输入密码（首次登录请修改）
3. 点击"登录"按钮

#### 密码策略
- 最小长度：8 位
- 必须包含：大写字母 + 小写字母 + 数字
- 建议包含：特殊字符

### 3. 仪表盘概览

登录成功后，自动进入仪表盘页面。

#### 仪表盘功能模块

| 模块 | 说明 |
|------|------|
| 运行总览 | 显示系统关键指标（会话数、Token 用量、活跃智能体） |
| Token 趋势 | 查看 Token 使用量变化（日/周/月） |
| 会话统计 | 显示活跃/已结束会话数量 |
| 实时监控 | 实时事件流（会话事件、智能体动作） |
| 分布统计 | Top 模型/渠道/工具分布图 |
| 智能体排行 | 按使用量排序的智能体列表 |

#### 数据下钻功能

点击任意统计卡片，可查看详细数据：
- **按会话**: 查看具体会话列表
- **按智能体**: 查看智能体使用详情
- **按渠道**: 查看各渠道使用分布
- **按模型**: 查看模型使用统计

## 核心功能详解

### 1. 在线对话 (Chat)

#### 功能说明
提供与 AI 智能体的实时聊天交互界面，支持多轮对话、技能调用、文件上传。

#### 操作指南

1. **选择智能体**
   - 从下拉框选择目标智能体
   - 或创建新智能体

2. **发送消息**
   - 输入消息内容
   - 按 Enter 发送
   - 按 Shift+Enter 换行

3. **查看回复**
   - AI 回复实时显示
   - 支持代码高亮
   - 支持 Markdown 渲染

#### 斜杠命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `/new` | 创建新会话 | `/new` |
| `/skill` | 查看可用技能 | `/skill` |
| `/model` | 切换模型 | `/model qwen-max` |
| `/status` | 查看会话状态 | `/status` |
| `/subagents` | 查看子智能体 | `/subagents` |

#### 常用语快捷回复

- 点击输入框上方的常用语按钮
- 选择预设消息快速发送
- 支持自定义常用语

### 2. 会话管理 (Sessions)

#### 功能说明
管理和监控所有会话，包括创建、重置、删除、导出等操作。

#### 会话列表操作

1. **筛选会话**
   - 按智能体名称筛选
   - 按状态筛选（活跃/已结束）
   - 按时间范围筛选

2. **会话操作**
   - **查看详情**: 点击会话进入详情
   - **重置会话**: 清除会话历史
   - **删除会话**: 永久删除会话
   - **导出记录**: 导出为 JSON/Markdown

3. **批量操作**
   - 批量删除会话
   - 批量重置会话
   - 批量导出会话

### 3. 记忆管理 (Memory)

#### 功能说明
管理智能体的记忆文档，这是智能体的"大脑"，定义了其身份、能力、行为准则。

#### 核心文档类型

| 文档名 | 说明 | 示例内容 |
|--------|------|---------|
| `AGENTS.md` | 角色定位和职责 | "你是软件开发团队团队组长..." |
| `SOUL.md` | 人格内核和思维模式 | "核心人格特质、语言风格..." |
| `IDENTITY.md` | 身份信息 | "Name: 软件开发团队团队组长..." |
| `USER.md` | 用户画像 | "用户基本信息、偏好..." |
| `TOOLS.md` | 工具使用指南 | "工具列表、使用规则..." |

#### 编辑文档

1. 选择智能体
2. 选择要编辑的文档
3. 在 Markdown 编辑器中修改内容
4. 点击"保存"按钮

#### 模板片段

- 点击工具栏的模板按钮
- 插入常用片段（如角色定义、技能描述）

### 4. 任务计划 (Cron)

#### 功能说明
创建和管理定时任务，支持多种触发方式。

#### 触发方式

| 方式 | 说明 | 示例 |
|------|------|------|
| Cron 表达式 | 标准 Cron 语法 | `0 9 * * *` (每天 9 点) |
| 固定间隔 | 每隔 N 分钟/小时 | `每 30 分钟` |
| 指定时间 | 一次性执行 | `2026-04-12 10:00` |

#### Cron 可视化编辑器

1. 选择频率（每天/每周/每月）
2. 设置具体时间
3. 实时预览 Cron 表达式
4. 一键复制到任务配置

#### 任务模板

- **晨报模板**: 每天早上 9 点发送日报
- **健康巡检**: 每小时检查系统状态
- **周报模板**: 每周一发送周报
- **自定义模板**: 根据需求创建

### 5. 模型管理 (Models)

#### 功能说明
配置和管理 AI 模型，支持多模型渠道。

#### 支持的模型渠道

| 渠道 | 说明 | 状态 |
|------|------|------|
| Qwen (通义千问) | 阿里云模型 | ✅ |
| Claude | Anthropic 模型 | ✅ |
| GPT (OpenAI) | OpenAI 模型 | ✅ |
| Gemini | Google 模型 | ✅ |
| 本地模型 | Ollama/vLLM 等 | ✅ |

#### 添加模型步骤

1. 点击"添加模型"按钮
2. 选择模型渠道
3. 填写 API Key（安全存储）
4. 配置模型参数（温度、最大 Token 等）
5. 点击"探测"测试连接
6. 保存配置

#### 模型套餐

可配置模型套餐，如：
- **Coding 套餐**: 适合代码生成
- **Chat 套餐**: 适合对话交互
- **Analysis 套餐**: 适合数据分析

### 6. 频道管理 (Channels)

#### 功能说明
配置和管理多渠道支持，让 AI 智能体可以通过多种方式与用户交互。

#### 支持的渠道

| 渠道 | 配置方式 | 说明 |
|------|---------|------|
| QQ | 机器人 + 私有部署 | 支持群聊/私聊 |
| 飞书 | 企业应用 | 支持群聊/单聊/审批 |
| 钉钉 | 企业应用 | 支持群聊/单聊 |
| 企业微信 | 企业应用 | 支持群聊/单聊 |

#### 飞书频道配置示例

1. 在飞书开放平台创建应用
2. 获取 App ID 和 App Secret
3. 配置事件订阅回调
4. 在 OpenClaw-Admin 中填写配置
5. 启用频道

## 高级功能

### 1. 多智能体协作 (Office)

#### 功能说明
创建多智能体协作场景，实现复杂任务的分工协作。

#### 场景类型

| 场景 | 说明 | 适用 |
|------|------|------|
| 软件开发团队 | 产品经理 + 架构师 + 开发 + 测试 | 软件开发 |
| 内容创作团队 | 策划 + 文案 + 设计 + 审核 | 内容生产 |
| 客服团队 | 接待 + 技术 + 售后 | 客户服务 |

#### 协作流程

1. 创建场景
2. 配置场景中的智能体
3. 定义协作规则
4. 委派任务
5. 跟踪进度
6. 查看结果

### 2. 虚拟公司 (MyWorld)

#### 功能说明
可视化办公场景，支持角色移动和互动。

#### 功能特点

- 3D 办公场景
- 角色移动（鼠标/键盘）
- 区域互动（工位、会议室、休息区）
- 实时通信

### 3. RBAC 权限管理

#### 功能说明
基于角色的访问控制，实现细粒度的权限管理。

#### 权限模型

```
用户 → 角色 → 权限
```

#### 内置角色

| 角色 | 说明 | 权限范围 |
|------|------|---------|
| Admin | 系统管理员 | 全部权限 |
| Operator | 操作员 | 大部分操作权限 |
| Viewer | 查看者 | 只读权限 |

#### 自定义权限

可创建自定义权限，如：
- `sessions:batch_delete` - 批量删除会话
- `models:export` - 导出模型配置
- `audit:view` - 查看审计日志

### 4. 通知中心

#### 功能说明
系统通知和告警，支持多渠道推送。

#### 通知类型

| 类型 | 触发条件 | 推送渠道 |
|------|---------|---------|
| 系统通知 | 系统事件 | 站内信 |
| 告警通知 | 异常检测 | 邮件/短信/飞书 |
| 任务通知 | 定时任务完成 | 站内信/邮件 |

#### 告警渠道

- 邮件
- 短信
- 飞书群机器人
- 钉钉群机器人
- Webhook

## 常见问题

### Q1: 无法登录系统

**可能原因**:
1. 用户名或密码错误
2. Gateway 连接异常
3. 浏览器缓存问题

**解决方案**:
1. 检查用户名密码是否正确
2. 检查 Gateway 服务是否运行 (`pm2 status`)
3. 清除浏览器缓存后重试

### Q2: 会话创建失败

**可能原因**:
1. 模型配置错误
2. 网络连接问题
3. Gateway 服务异常

**解决方案**:
1. 检查模型配置是否正确（API Key 是否有效）
2. 检查网络连接
3. 查看系统日志 (`pm2 logs openclaw-admin`)

### Q3: 定时任务不执行

**可能原因**:
1. Cron 表达式错误
2. 任务被禁用
3. 系统时间不正确

**解决方案**:
1. 使用可视化编辑器验证表达式
2. 检查任务状态是否为启用
3. 检查服务器时间 (`date`)

### Q4: 模型探测失败

**可能原因**:
1. API Key 无效
2. 网络连接问题
3. 模型渠道配置错误

**解决方案**:
1. 检查 API Key 是否正确
2. 检查网络连接（curl 测试）
3. 确认模型渠道配置

### Q5: 权限不足

**可能原因**:
1. 用户角色权限不够
2. 权限未正确分配

**解决方案**:
1. 联系管理员提升权限
2. 检查角色权限配置
